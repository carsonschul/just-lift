Just Lift — Solver Notes (v1)

The questions


- Q1: days/week (1–6)
- Q2: session length (30 / 45 / 60 / 90+ min)
- Q3: experience (beginner / experienced)


Volume


- 6–12 sets per muscle per week
- beginner → lean ~6-8, experienced → lean ~10–12
- fractional counting: direct set = 1.0, secondary muscle = 0.5
- muscle tiering: compound-rich muscles (chest, back, quads, hams) can hit the top
of the range; small/isolation muscles (delts, arms, calves) cap lower (~6) — that's
a realistic ceiling, not a shortfall (anatomy, not a programming failure)


Effort


- default: near failure (failure to 2 RIR assumed)
- disclosed on the plan


Rest


- 4 min for compounds, 2 min for isolation (generous to respect user time, can be customized by user)


Sets per exercise


- 2–3 (can be customized by user)


Splits (by days)


- 1–3 → full body
- 4 → upper/lower
- 5 → PPL + Upper/Lower (default)
- 6 → PPL
- (5-day alt: U/L + extra day — user selection option)


Time budget


- reserve 15 min overhead (setup, transitions, walking, waiting)
- fill the rest with lift + rest at upper-end costing
- add exercises until the session fills


Exercise selection


- compound-first (best coverage per minute)
- common movements only (familiarity bias baked in — no obscure lifts)


Progression


- double progression, single target
- 12, 12 default target for 2 sets
- 8, 8, 8 default target for 3 sets
- progression scheme can be customized by user


Output


- the plan + one line explaining its reasoning ("this plan assumes X rest / Y effort")



Later (not v1)


- swaps ("gym doesn't have this" / "replace this") — schema already supports it
- shorter rest, more sets options (allow user to allocate time manually)
- split override (U/L+extra vs PPL+U/L, etc.)
- compounds-only toggle
- tracking / evolving plans / memory — the PAID tier
- animated plan rebuild on any change


Product philosophy


- default = a sane, opinionated plan. advanced users click to override.
- obey instantly, explain transparently, never argue
- free = the generator (reacting). paid = memory (remembering you over time).
- plausible > optimal. be real with the user
